# 终结 Electron 的“内存霸权”？Pake：5MB 打包任意网页，让你的桌面重获新生

大家好，我是太阳鸟。

作为一名资深开发者，我每天都要在数十个浏览器标签页中穿梭。

微信读书、DeepSeek、ChatGPT、Excalidraw……这些工具虽然好用，但它们被困在浏览器里，不仅管理难，更重要的是：**它们极易分散注意力。**

我曾尝试寻找这些工具的桌面客户端，但现实很残酷：
1.  **根本没有**：很多产品只有 Web 版。
2.  **胖得吓人**：有些客户端基于 Electron 开发，动辄 100MB+ 的安装包，几十个进程吃掉我数 GB 的 RAM。

直到我遇到了 **Pake**。

---

### 1. 什么是 Pake？

**Pake**（由开发者 tw93 创建）是一个极简的开源工具，它能让你用**一行命令**就把任何网页变成一个独立的桌面应用。

-   **项目主页**：https://github.com/tw93/Pake
-   **核心定位**：极轻量（~5MB）、极快、跨平台（macOS, Windows, Linux）。

如果你想把微信读书变成一个有独立图标 and 窗口的桌面应用，用 Pake 只需要几秒钟。

---

### 2. 技术专家的视角：为什么 Pake 优于 Electron？

作为技术专家，我们必须看底层架构。Pake 成功的核心在于它抛弃了臃肿的 Electron，转而拥抱了 **Tauri**。

#### **对比架构图 (Mental Model):**

![Pake vs Electron 体积对比](/images/pake/infographic.png)

| 维度 | Electron | Pake (Tauri) |
| :--- | :--- | :--- |
| **运行时 (Runtime)** | 内置完整的 Chromium 引擎 | 调用系统原生的 Webview (WebKit/Edge) |
| **后端语言** | Node.js | Rust (高效、内存安全) |
| **安装包大小** | 通常 100MB 起步 | **约 5MB (缩减 20 倍)** |
| **内存占用** | 极高 (每个应用一个独立引擎) | 极低 (共享系统内核) |

**这就是所谓的“降维打击”。** Pake 通过调用系统原生的渲染引擎，实现了极致的轻量化。同时，后端使用 Rust 编写，保证了极高的运行效率和响应速度。

---

### 3. Pake 的“杀手锏”功能

除了“小”，Pake 还做到了“强”：

1.  **沉浸式窗口**：支持隐藏标题栏，让应用看起来更像是一个原生 APP。
2.  **快捷键支持**：你可以为网页定义桌面级的快捷键，提升操作效率。
3.  **样式定制**：通过注入自定义 CSS/JS，你可以去掉网页广告、修改主题颜色，定制属于你自己的“洁净版”工具。
4.  **智能提取**：打包时能自动抓取网站 Icon，并生成对应的应用图标。

---

### 4. 实践：三分钟拥有你的专属 App

![Pake 桌面应用效果演示](/images/pake/mockup.png)

#### **开发者：一行冷酷的 CLI 命令**
如果你本地有 Node.js 环境，直接运行：

```bash
# 安装命令行工具
pnpm install -g pake-cli

# 打包 GitHub (带自定义图标和窗口大小)
pake https://github.com --name GitHub --width 1200 --height 800
```

#### **小白用户：在线打包，彻底零门槛**
Pake 非常贴心地利用了 **GitHub Actions**。你只需要 Fork 一个仓库，在网页上输入 URL 和名字，GitHub 就会在服务器上帮你打包好，你直接下载成果文件（.dmg / .exe）即可。

#### **懒人福利：流行包直接下**
作者已经预打包了很多常用工具：
-   **WeRead (微信读书)**：沉浸感极强。
-   **DeepSeek / ChatGPT**：AI 时代必备的独立窗口。
-   **YouTube Music**：让网页音乐变成你的桌面播放器。
