## 一、Open Code 组合到底牛在哪？

简单来说，这是一套**完全开源可定制的 AI Agent 流水线系统**！

它的厉害之处在于两点：

### 1.1 全自动拆解任务的「智能工厂」

你只要在左侧给**大 Agent（主 Agent）**下达需求，它会自动分析、拆解、分配给不同的子 Agent 去执行。

**举个例子：**

你要写一篇深度科技稿，主 Agent 会自动：
- 派**调研 Agent** 查数据
- 派**逻辑梳理 Agent** 搭框架
- 派**写作 Agent** 写初稿
- 派**优化 Agent** 润色内容
- 派**审核 Agent** 最终检查

后台一堆任务同时跑，**效率直接拉满！**

### 1.2 彻底向你开放的「改装平台」

普通的 AI 工具就像「量产车」，只能给你几个固定的按钮调节。

而 **Open Code 是「赛车改装厂」**：
- 从模型选择到 Agent 配置
- 甚至创建全新的 Agent
- 所有细节都向你展开了！

你可以：
- ✅ 通过配置文件做小调整（比如给某个 Agent 加专属提示词）
- ✅ 把整个项目"扒"下来，重新设计符合自己工作习惯的 Agent 体系

## 二、先搞懂这 2 个核心配置逻辑，才能玩明白

在动手定制之前，你必须先区分**用户级配置**和**项目级配置**，不然改了半天可能还不生效！

### 2.1 用户级配置（默认全局设置）

安装好 Open Code 后，在电脑的 `config` 文件夹下会有一个专门的 Open Code 文件夹。

**这里的配置是所有项目默认遵循的"家规"**，除非项目里有特别设定。

```
用户目录/config/OpenCode/
├── opencode.json
└── open-my-opencode.json
```

### 2.2 项目级配置（单独项目专属）

在每个项目的根目录，你可以创建一个 `.opencode` 的隐藏文件夹。

**这里的配置优先级高于用户级**，比如你可以给某个科技项目单独配置调研能力更强的 Agent。

```
项目根目录/.opencode/
├── opencode.json
└── open-my-opencode.json
```

### 2.3 核心配置文件

这两个层级的文件夹里，都有 **2 个最重要的文件**：

| 文件名 | 作用 |
|--------|------|
| `opencode.json` | 配置 Open Code 本体（模型选择、插件配置） |
| `open-my-opencode.json` | 配置 My Open Code 插件（Agent 协作规则） |

**配置优先级：**

```
项目级配置 > 用户级配置
```

如果项目级没有配置，就使用用户级的默认配置。

## 三、我为内容创作定制的专属插件：New Type Profile

### 3.1 为什么要定制？

因为我平时做的更多是**深度内容创作**，而原版 My Open Code 插件里的 Agent 都是为编程场景硬编码的（不能改基础名称和提示词），用起来总觉得"不对味儿"。

**所以我把整个插件扒下来，专门为内容场景重新设计了 8 个 Agent！**

### 3.2 8 个内容创作专属 Agent

| Agent 名称 | 功能 |
|-----------|------|
| **调研 Agent** | 搜索资料、验证数据来源 |
| **逻辑梳理 Agent** | 搭建文章框架、理清思路 |
| **写作 Agent** | 撰写初稿、扩展内容 |
| **数据分析 Agent** | 处理数据、生成图表 |
| **验证 Agent** | 事实核查、逻辑验证 |
| **优化 Agent** | 润色文字、提升可读性 |
| **格式化 Agent** | 调整排版、添加标记 |
| **审核 Agent** | 最终检查、质量把关 |

### 3.3 最牛的优化：主 Agent 自动切换「探讨」和「执行」模式

这个自动切换有多香？举个真实例子：

**场景：** 我问 AI "英伟达的算力卡故障率约 9%，这个数据来源和真实性如何？"

**传统 Agent 的做法：**
- 直接给你一个"真"或"假"的答案
- 或者简单搜索一下就回复

**New Type Profile 的做法：**

1. **主 Agent 首先判断**：这个需求是**「探讨」而非执行**（需要先理清楚上下文，而不是直接写报告）
   - 自动进入「探讨伙伴」模式

2. **接着拆解任务**：
   - 派 1 个**调研 Agent** 查数据来源
   - 派 1 个**验证 Agent** 找行业佐证

3. **最后给出结论**：
   - 不是简单说"真"或"假"
   - 而是告诉了我这个观点的**完整上下文**
   - 比如是哪个研究机构统计的、针对什么场景
   - 以及引发行业讨论的核心原因

**这就是「探讨模式」的价值！**

## 四、超级详细的安装&使用指南

### 第一步：安装 Open Code 本体

1. 打开 GitHub，搜索「Open Code」（注意区分大小写）
2. 按照官方文档的指引下载安装包
3. 安装完成后，打开软件并登录账号（支持多种登录方式）

### 第二步：安装 My Open Code 插件

1. 在 Open Code 的插件市场里搜索「My Open Code」
2. 点击「安装」，等待安装完成
3. 安装成功后，插件会自动生效

### 第三步：安装我的内容创作专属插件

**（悄悄说：这个插件已经发布到 GitHub 上了，搜索关键词「new type profile」就能找到！）**

1. 打开 GitHub，搜索「new type profile」
2. 下载项目压缩包，解压到电脑的某个文件夹
3. 打开 Open Code，点击「插件管理」→「本地插件」→「添加插件」
4. 选择刚才解压的文件夹，点击「确定」
5. 插件安装成功后，在「插件设置」里启用它

### 第四步：配置和使用

**配置用户级默认设置：**

```bash
# 找到用户配置目录
用户目录/config/OpenCode/

# 编辑 opencode.json
{
  "model": "gpt-4",  // 选择你的模型
  "plugins": ["my-opencode", "new-type-profile"]  // 启用插件
}
```

**配置项目级专属设置：**

```bash
# 在项目根目录创建
项目根目录/.opencode/

# 编辑 open-my-opencode.json
{
  "agents": {
    "research": {
      "prompt": "你是一个专业的调研 Agent..."
    },
    "writing": {
      "prompt": "你是一个擅长写作的 Agent..."
    }
  }
}
```
