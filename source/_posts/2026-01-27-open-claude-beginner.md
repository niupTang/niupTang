## 一、 为什么小白更该用命令行工具？

很多人觉得写代码得有个华丽的界面。但真相是：**图形界面（GUI）往往会分散你的注意力。**

命令行工具（CLI）最大的优势在于：**响应快、没废话、直控文件系统。**

### OpenCode 的「傻瓜式」体验：
1. **不用配置复杂的环境变量**：装上就能开跑。
2. **多模型自由切换**：你可以用 Claude 3.5 辅助写逻辑，用 Gemini 刷代码注释，甚至用本地大模型搞基建。
3. **沉浸式对话**：它提供了一个非常酷的 TUI（终端用户界面），就像在玩一款复古文字游戏，但它能帮你写出跑得飞快的代码。

## 二、 三分钟，开启你的第一个 AI Agent

别被「命令行」吓跑，跟我做这几步，你就能拥有一个属于自己的「AI 程序员」。

### 2.1 安装（一键直达）
不管你是 Mac 还是 Windows，只要一行命令（需要 Node.js 环境）：

```bash
npm install -g @opencode-ai/cli
```

### 2.2 给你的项目「开个会」
进入你的项目文件夹，输入：

```bash
opencode --init
```

它会自动分析你的项目结构，甚至会贴心地提醒你：**「嘿，你的 README 还没写，要我帮忙吗？」**

## 三、 核心必杀技：My Open Code 插件

为什么我特别推荐这套组合？因为它能让小白瞬间变身「架构师」。

这套工具支持通过 `AGENTS.md` 文件**定义 AI 的个性**。你可以告诉它：
- 「你是一个擅长 Python 数据分析的老司机。」
- 「你是一个对前端 UI 极度挑剔的设计师。」

当你下达指令时，这些「子 Agent」会自动协作，比你一个一个去改 Prompt 效率高出不止一个量级！

## 四、 牛皮糖的小撇步：如何优雅地避坑？

1. **别一次塞太多文件**：AI 的「胃口」（上下文）有限。小白建议先从小模块开始，循序渐进。
2. **多用 `/vibe` 模式**：如果你感觉思路卡住了，直接让 AI 给你提建议，别自己硬抠代码。
3. **活用 Git**：命令行配合 Git 就是无敌的备份方案。万一 AI 把代码写乱了，一键 `git reset` 就能重来，毫无心理负担！
