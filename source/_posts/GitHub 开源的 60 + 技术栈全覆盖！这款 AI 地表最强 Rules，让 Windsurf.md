这些文件夹主要包含与不同技术栈、开发场景相关的 `.cursorrules` 配置文件及说明文档，用于定制 Windsurf 编辑器的 AI 代码生成规则。具体内容可分为以下几类：

### 1. **技术栈专属规则**
针对特定编程语言、框架或工具，定义代码生成的风格、规范和最佳实践：

- • **前端技术**：如 `nextjs-typescript-tailwind-cursorrules-prompt-file/`（Next.js + TypeScript + Tailwind）、`sveltekit-restful-api-tailwind-css-cursorrules-pro/`（SvelteKit + Tailwind）、`vue-3-nuxt-3-typescript-cursorrules-prompt-file/`（Vue 3 + Nuxt 3）等，包含框架特定的项目结构、组件设计、状态管理等规则。

- • **后端技术**：如 `nodejs-mongodb-jwt-express-react-cursorrules-promp/`（Node.js + MongoDB + JWT）、`go-servemux-rest-api-cursorrules-prompt-file/`（Go 语言 REST API）等，涉及数据库交互、认证授权、API 设计规范。

- • **编程语言**：如 `python-containerization-cursorrules-prompt-file/`（Python 容器化）、`typescript-clasp-cursorrules-prompt-file/`（TypeScript + Google Apps Script）等，涵盖语言特性、代码风格、依赖管理规则。

### 2. **开发场景与工具集成**
针对特定开发场景（如 AI 应用、桌面应用、浏览器扩展）或工具链，提供定制化规则：

- • **AI 与数据科学**：如 `pytorch-scikit-learn-cursorrules-prompt-file/`（PyTorch + Scikit-learn）、`pandas-scikit-learn-guide-cursorrules-prompt-file/`（数据处理与分析），包含模型训练、数据可视化、特征工程的规范。

- • **桌面与扩展开发**：如 `tauri-svelte-typescript-guide-cursorrules-prompt-f/`（Tauri 桌面应用）、`chrome-extension-dev-js-typescript-cursorrules-pro/`（Chrome 扩展），涉及跨平台适配、API 调用、权限管理规则。

- • **DevOps 与工具链**：如 `python-github-setup-cursorrules-prompt-file/`（GitHub 项目配置）、`kubernetes-mkdocs-documentation-cursorrules-prompt/`（K8s 文档生成），包含 CI/CD 流程、版本控制、文档规范。

### 3. **通用开发规范**
定义跨项目的通用规则，确保代码质量和一致性：

- • **设计原则**：如 `optimize-dry-solid-principles-cursorrules-prompt-f/`（DRY、SOLID 原则），指导代码复用、模块化和可维护性。

- • **代码风格**：如 `typescript-code-convention-cursorrules-prompt-file/`（TypeScript 编码规范）、`code-guidelines-cursorrules-prompt-file/`（通用代码指南），规定命名规则、注释风格、格式标准。

- • **项目管理**：如 `github-cursorrules-prompt-file-instructions/`（GitHub 协作规范），包含提交信息格式、分支管理、PR 流程等。

### 4. **文件结构与内容格式**
每个文件夹通常包含：

- • 核心配置文件：`.windsurfrules` 或 `global_rules.md`，定义 AI 生成代码的具体规则（如框架偏好、安全要求、输出格式）。

- • 说明文档：`README.md`，介绍规则适用场景、可构建的项目类型（如 “AI 模型压缩服务”“React 组件库管理器”）、使用方法及作者信息。

这些内容共同构成了针对不同技术场景的定制化规则库，帮助开发者通过 Windsurf 编辑器生成符合项目需求和团队规范的代码。
